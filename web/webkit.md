# WebKit

## 웹킷(WebKit)이란?

- 애플에서 **오픈소스**로 개발하고 있는 **웹 컨텐트(Web Content) 엔진** 또는 **웹 렌더링(Web Rendering) 엔진**이다.
- **웹 컨텐트**와 **웹 렌더링**은 웹킷의 기능을 나타내며, **엔진**과 **오픈소스**는 웹킷의 특성을 나타낸다.

### **엔진**으로써의 웹킷은?

- 일반적으로 엔진이라 하면 에너지를 변화시켜 자동차와 같은 기계장치에 동력을 제공해 주는 핵심부품을 말한다.
- 자동차의 엔진은 일반 고객이 쉽게 접하지 못하는 부품이나, 자동차의 특성과 성능에 큰 영향을 미친다.
- 이처럼 소프트웨어 엔진도 웹 브라우저의 특성에 큰 영향을 미치는 핵심요소로써, 웹 컨텐트 엔진의 특성에 따라 사용자가 경험하는 웹의 세상은 큰 차이를 보이게 딘다.
- 정리하자면 웹킷은 **웹 브라우저** 또는 이메일이나 위젯과 같은 **웹 관련 애플리케이션**의 핵심 부품 역할을 하는 소프트웨어 엔진이다.

### **오픈소스**의 특성은?

- 소스코드가 공개되어 있어 사용자가 자유롭게 가져다가 사용해 볼 수 있으며 수정, 재배포도 가능하다.
- 반면, 제품으로서의 품질을 보장해주지 않는다.
- 따라서 사용자(개발자)가 오픈소스를 이용하려면 오픈소스 프로젝트의 목표를 정확히 이해해야 한다.
- 정리하자면 웹킷을 하나의 완성품으로 보고 쉽게 상업용 제품 개발에 이용하는 것은 조심스런 주의가 필요하다.

### 웹 컨텐트를 렌더링한다는 것은?

- 웹 컨텐트는 웹 상에 무수히 존재하는 컨텐트(HTML, CSS, JavaScript)를 말한다.
- 웹킷은 이러한 웹 컨텐트를 사용자의 출력장치에 표현하고 사용자와 상호작용하는 기능을 가진다.
- 정리하자면 웹 브라우저에서의 렌더링이라 함은 HTML 문서를 CSS에 명시된 속성으로 **출력장치에 배치**하는 작업을 말한다.

즉, 웹킷은 HTML, CSS, JavaScript로 이루어진 웹 컨텐트를 사용자 출력장치에 표현(*배치, 렌더링*)하고 사용자와 상호작용하는 웹 애플리케이션 소프트웨어의 핵심부품이다.

### 웹킷 오픈소스 프로젝트의 목표

WebKit.org에 기술되어 있는 [프로젝트 목표](https://webkit.org/project/)는 다음과 같다.

- **웹 컨텐트 엔진**
    - 프로젝트의 최우선적인 대상은 HTML, CSS, JavaScript, 그리고 DOM과 같은 표준 웹 기술을 사용해 작성된 www 상의 컨텐트이다.
    - 다른 웹 관련 애플리케이션에서 웹킷을 이용할 수 있도록 해서 사용자에게 웹 컨텐트를 보여주거나 상호작용하는 엔진이 되기를 원한다.
- **오픈소스**
    - 다른 오픈소스나 독점적인 애플리케이션 모두에서 자유롭게 사용될 수 있도록 해야 한다.
    - 특히, LGPL 2.1+와 호환되는 라이센싱을 목표로 하고 있다. 현재로서는 웹킷 라이센스를 LGPL 3으로 옮길 계획은 없다.
- **호환성**
    - 호환성은 수많은 기존 사이트에게 있어서 필수적인 요소이다.
    - 표준 준수 비용이 많이 들더라도 기존 웹사이트와의 호환성을 유지하고 개선시키도록 노력하고 있다.
- **표준 준수**
    - 웹 표준을 준수하고 새로운 표준을 지원하는데 목표를 두고 있다.
    - 실제로 웹 표준화 커뮤니티에 참여하면서 새로운 기술들을 표준에 반영하고 새로운 표준을 엔진에 구현하여 표준의 타당성을 검사하고 있다.
- **안전성**
    - 웹킷 코드는 최고 수준의 안정성을 보장하기 위해 관리되고 있다.
    - 엔진이 죽거나, 응답이 없거나, 하위 호환성 문제가 발생할 경우 즉각적으로 처리된다.
- **성능**
    - 속도와 메모리 사용량을 개선하고 관리하는 것도 중요한 목표이다.
- **보안**
    - 보안 이슈를 즉각적으로 해결하여 사용자를 보호하고 신뢰를 유지하고 있다.
- **이식성**
    - 웹킷을 다양한 데스크탑, 모바일, 임베디드 등의 플랫폼에 포팅이 용이하도록 하고 있다.
- **사용성**
    - 좋은 사용자 인터페이스 설계 원칙에 따라 작업하고 있다.
- **분석 용이성**(Hackability)
    - 상대적으로 이해하기 쉬운 코드를 만들고자 노력하고 있다.

### 프로젝트의 범위를 벗어나는 부분

이 또한 WebKit.org에 기술되어 있다.

- 웹킷은 **엔진**이지, *브라우저*가 아니다.
    - 웹킷 기반의 완전한 브라우저를 개발할 계획이 없다.
    - 웹킷을 이용해 웹 브라우저를 개발하려면 캐시, 주소창, 북마크, UI 등의 상당히 많은 추가작업이 필요하며, 각종 기능의 포트를 해당 플랫폼에 맞게 작성하는 것도 상당히 많은 개발작업을 필요로 한다.
- 웹킷은 과학 프로젝트가 아닌 공학 프로젝트이다.
    - 새로운 기능을 적용하기 위해, 관련 기술이나 적어도 검증을 위해 use case를 더 선호한다.
- 웹킷은 재사용이 가능한 범용 코드로 구성되어 있지 않다.
    - 때론 범용 목적의 코드를 작성하지만, 좋은 웹 컨텐트 엔진을 위해 필요한 수준으로만 작성한다.
- 웹킷은 모든 문제에 대한 해결책이 아니다.
    - 웹킷은 웹 컨텐트에 집중한다. 상상할 수 있는 모든 기술에 대한 솔루션을 제공하려는 것이 아니다.
