# 스키마(schema)
## 스키마란?
- 데이터베이스의 구조와 제약 조건에 관한 전반적인 명세를 기술한 메타데이터의 집합
- 데이터베이스를 구성하는 데이터 개체(Entity), 속성(Attribute), 관계(Relationship) 및 데이터 조작 시 데이터 값들이 갖는 제약 조건 등에 관해 전반적으로 정의한다.
- 사용자의 관점에 따라 외부 스키마(external schema), 개념 스키마(conceptual schema), 내부 스키마(internal schema)로 나뉜다.


## 스키마의 3계층
![](https://t1.daumcdn.net/cfile/tistory/222516395942259029)

스키마는 3계층 스키마로 나누어져 구성되어 있고, 각각의 스키마는 결국 데이터베이스의 구조와 제약조건에 대한 전반적인 명세를 기술한 것을 의미한다. 3계층으로 나눈 이유는 관점에 따라서 분류한 것이다.

1. **외부 스키마**
- 개인의 입장, 즉 사용자 뷰(View). 서브 스키마(Sub Schema)라고도 한다.
- 하나의 외부스키마는 여럿이 공유 가능하다.
- 하나의 DB시스템에 여러 개의 외부스키마가 존재 가능하다.

2. **내부 스키마**
- 시스템 프로그래머나 설계자의 관점에서 바라보는 스키마, 즉 저장 스키마(Storage Schema)이다.
- 데이터베이스의 물리적 구조이다.

3. **개념 스키마**
- 조직 전체의 입장, 즉 전체적인 뷰(View)이다.
- 개체 간의 관계와 제약조건을 나타낸다.
- 데이터베이스의 접근권한 / 보안 / 무결성 규칙에 대한 명세를 정의한다.

>  일반적으로 '스키마'는 내부 스키마(Internal Schema)를 말한다.
