# X.25 프로토콜
DTE와 DCE간의 인터페이스를 제공, 패킷 교환망을 통해 패킷을 원활히 전달하기 위한 통신 절차
> - DTE(Data Terminal Equipment, 데이터 터미널 장치)
> - DCE(Data Circuit-terminating Equipment, 데이터 회선 종단장치)

## 특징
- 연결형 프로토콜 흐름제어, 오류 제어 등의 기능이 있다.
- 강력한 오류 체크 기능으로 신뢰성이 높다.
- 한 회선에 장애가 발생해도 우회 전송 가능하다.
- 가상 회선 방식을 이용해서 효율성이 높다.
- 축적 교환방식으로 처리 지연이 발생할 수 있다. (단점)

## 계층 구조
- 물리 계층 - 물리적 접촉 인터페이스로 X.21 사용
- 프레임 계층 - 데이터 링크 계층에 해당한다. 다중화, 순서 제어, 오류 제어, 흐름 제어 등, LAPB 사용
- 패킷 계층 - 네트워크 계층에 해당

X.25는 일반적으로 OSI 7계층에서 네트워크 계층에 해당하는 프로토콜로 알려져 있지만, 위 계층구조에서 알 수 있듯이 데이터링크 계층과 물리 계층까지 포함한다.
